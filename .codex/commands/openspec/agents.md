# OpenSpec Agent Rules for edupen

## Tech stack
- Flutter
- GetX for bindings/controllers
- Keep UI code under lib/pages/*
- Reuse existing widgets before creating new ones

## Coding rules
- Prefer minimal changes
- Do not refactor unrelated modules
- Keep naming consistent with existing sign_in files
- Avoid adding dependencies unless required

## Validation rules
- Run flutter analyze after changes when possible
- Do not mark tasks done before verification
